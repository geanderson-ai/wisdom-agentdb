# UI Spec

## Product direction

The first interface should feel like a memory control center, not a chat toy.

## Primary surfaces

- Timeline of captured memories
- Search and recall panel
- Project and session scope switcher
- Privacy and retention controls
- Citation and source trace view

## Visual direction

- Dark-first developer UI
- High-contrast type
- Dense information layout
- Clear provenance indicators
- Minimal visual noise

## Core interactions

- Save a memory from a session
- Search by meaning
- Inspect source and citations
- Mark memory private or deletable
- Reinject a memory into the current context

## UX rule

- Users should be able to see what the system knows, where it came from, and why it is being used.
